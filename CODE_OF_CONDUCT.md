# Code of Conduct

## Our Pledge

Contributors and maintainers should make participation in CAP respectful, constructive, and professional for everyone.

## Expected Behavior

Please:
- be respectful in technical disagreement
- keep feedback specific and actionable
- prefer evidence, RFC references, and concrete proposals over dismissive argument
- help preserve the repository's docs-first workflow and clear decision boundaries

## Unacceptable Behavior

Examples of unacceptable behavior include:
- harassment or personal attacks
- discriminatory or demeaning language
- sustained disruptive behavior
- deliberately misrepresenting repository decisions or source-of-truth boundaries

## Scope

This code of conduct applies to repository discussions, issues, pull requests, and other project collaboration spaces.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers through the repository's private reporting path described in `SECURITY.md` or another maintainer channel made available by the project.

## Enforcement

Project maintainers may remove, edit, or reject contributions and participation that do not align with this code of conduct.
