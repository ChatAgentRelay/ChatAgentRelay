The Problem

Every AI chat integration is a custom spaghetti

You build Slack webhook → your code → OpenAI → Slack API. It works. Then your PM wants Teams. Compliance wants audit logs. The CEO switches to Claude. Each change is a rewrite. Each platform leaks into your logic.

The Solution

One canonical event chain, infinite combinations

Every message flows through a 7-event chain. Channels and agents are pluggable. Swap either side without touching business logic.

1 → message.received  user's message, canonicalized from any platform
2 → policy.decision.made  allow or deny (configurable rules)
3 → route.decision.made  select the right agent backend
4 → agent.invocation.requested  dispatch to AI backend
5 → agent.response.completed  agent's reply captured
6 → message.send.requested  queued for delivery with retry
7 → message.sent  delivered to user's chat platform
× → event.blocked  captures failures with reason + stage
What You Get

Built for the real world, not just demos

$ plug --channel

Any Chat Platform

Slack and WebChat built-in. Add Discord, Teams, or Telegram with one interface. No platform lock-in.

$ plug --agent

Any AI Agent

OpenAI and generic HTTP included. Add Claude, Gemini, or your custom model. Swap backends in one line.

$ cap policy

Governance Built-In

Keyword/regex policy engine blocks messages before they reach your agent. Every decision logged.

$ cap audit

Full Audit Trail

Every message produces an immutable event chain. Query conversations, correlations, and audit explanations via REST API.

$ cap stream

Streaming Responses

OpenAI SSE streams update Slack messages progressively. Users see the agent thinking in real-time.

$ cap retry

Delivery Retry + DLQ

Exponential backoff on failures. Dead-letter queue for exhausted retries. No lost messages.
